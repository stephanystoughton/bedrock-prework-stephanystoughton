Prompt Library

Explain Like I’m New

When I’d use it:
I would use this when I encounter a QA concept I don’t understand and need a beginner-friendly explanation.

R (Role): You are a senior QA mentor teaching a complete beginner.

T (Task): Explain the concept of [insert concept here].

C (Context): I am new to QA and have only completed the first few modules of the course.

C (Constraints): Use simple language, avoid jargon when possible, and include one practical example.

O (Output Format): Provide a short explanation, one example, and three key takeaways.

Generate Test Cases for X

When I’d use it:
I would use this when I need ideas for testing a feature.

Role: You are an experienced QA analyst.

Task: Generate test cases for [feature].

Context: The feature is being tested during a manual QA cycle.

Constraints: Include positive, negative, and edge cases. Create at least 10 test cases.

Output Format: Present the results in a table with Test ID, Description, Steps, and Expected Result.

Review My Work

When I’d use it:
I would use this when I want feedback on a bug report or test case document.

Role: You are a QA team lead.

Task: Review the following work: [paste work here].

Context: This is a practice assignment from a QA course.

Constraints: Identify unclear wording, missing information, and improvement opportunities.

Output Format: List strengths first, then weaknesses, then recommendations.

Debug This Error

When I’d use it:
I would use this when my code or command line produces an error I do not understand.

Role: You are a QA automation mentor.

Task: Help me diagnose and fix this error.

Context: I am working on a beginner QA programming exercise.

Constraints: Explain the likely cause, suggest a fix, and explain how to avoid the mistake in the future.

Output Format: Show:

Possible cause
Recommended fix
Prevention tip

Error:
[paste error here]

Predicted AI Response

The AI would likely identify the most probable cause of the error, explain what part of the code or command is incorrect, and suggest a corrected version. It would probably also provide a brief explanation of why the error occurred and how to avoid making the same mistake again.

Act as a Senior QA

When I’d use it:
I would use this when I need guidance on how an experienced QA professional would approach a task.

Role: You are a senior QA engineer with 10 years of experience.

Task: Analyze the following feature and recommend a testing strategy.

Context: The feature is [describe feature here].

Constraints: Focus on risk areas, important test scenarios, and common failures.

Output Format: Include risk assessment, recommended tests, and testing priorities.

Verification Habits

For high-stakes claims, I will compare answers from at least two AI tools before trusting the information.
If an AI references a specific tool, version, or standard, I will verify that the version and details actually exist and are current.
If an AI references OWASP, ISTQB, or another authoritative source, I will check the official website or documentation before relying on the information.
